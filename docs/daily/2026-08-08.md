### IQA 找出 6 项 Medium

| # | 严重度 | 描述 | 触发场景 |
|---|---|---|---|
| **M1** | 🟡 Medium(真 BUG)| DAO `Value(null)` 覆盖已有数据 | 已存在 lend 账户二次记账 + UI 留空起始余额 → 历史 initialLendBalanceCents 被清 |
| **M2** | 🟡 Medium(真 BUG)| initialTime 在每次 lendMoney 被错误覆盖 | 已存在 lend 账户二次记账 + 起始时间 picker 选今天 → 账户 initialTime 被改成今天,语义破坏 |
| **M3** | 🟡 Medium | ADR-0030 §决策 2 字面 vs 实现偏离,ADR 未更新 | originalTransactionId 字面写 `.references(Transactions, #id)`,实际没加 FK |
| **M4** | 🟡 Medium | migration_v8_test 不验证真 v7→v8 路径 | 只验 fresh install,不验 onUpgrade 触发 |
| **M5** | 🟡 Medium | UI 重复解析 `_initialBalanceController.text` | `_save()` 调 2 次 |
| **M6** | 🟡 Medium | widget test 只覆盖 UI 元素存在,未覆盖全链路 | 验证不了 _save() → dao 落库 |

### 修复决策

用户拍板「修全部 6 项 Medium」。

**修复方案**:
1. **M1+M2**:DAO 删除 `initialLendBalanceCents` + `initialTime` 2 参数 + 删 updateAccountById 块;UI 继续在 insertAccount 路径写 accounts(新建一次,已存在账户不改)
2. **M3**:ADR-0030 §决策 2 字面改为「nullable 不加 FK」+ WHY(沿用 fromAccountId 策略)
3. **M4**:复制 migration_v4_test 模式,写 `migration_v8_upgrade_test.dart` file-based 真升级 4 用例
4. **M5**:`_save()` 顶部提取 `_parseCents(_initialBalanceController.text)` 一次
5. **M6**:折中 — DAO 全链路测试覆盖「_save 调用 dao 时参数正确」(在 `lend_borrow_test.dart` 加 2 用例),不再做端到端 widget test(ROI 低 + DatePicker/BottomSheet mock 复杂)
