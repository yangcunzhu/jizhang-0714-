## 🐛 遇到问题与解决

| 问题 | 解决 |
|---|---|
| account_form_provider 用 autoDispose → 测试触发 _debugIsMounted | 改非 autoDispose — family key 已天然分隔不同弹层实例 |
| copyWith 的 `null ?? existing` 语义无法"传 null 清空"信用卡字段 | 信用卡字段 setter 改用直接构造 AccountFormState,显式控制 null 语义 |
| accountListProvider stream emit 时序测试 flaky | 简化测试,只验证 "写入 → 重新订阅拿到新数据",不强求 emit 顺序 |
| Drift StreamProvider 在 fake_async 下 dispose hang(测试 did not complete) | 改 ProviderContainer + UncontrolledProviderScope 显式控制生命周期 + tearDown 先 dispose container 再 close db |
| tester.pump() 不触发 Drift 真实 emit | 用 `tester.runAsync(() => Future.delayed(50ms))` 让 Drift 走真 timer |
| ListTile 在 Container/DecoratedBox 内有 background color → Flutter 警告 | 弹层顶层 Container 改 Material(color: ... + borderRadius: ...) |
| `添加账户` 同时出现在 FAB label 和弹层标题 → findsOneWidget 撞 | 改 findsAtLeastNWidgets(1) |
| AccountEntry 显示 2 处(name + type.displayName)→ findsOneWidget 撞 | 改 findsWidgets(≥1) |
| `复制新账户` 类型限制时仅切到信用卡字段 creditLimit 为 null → 整个信用卡字段不显示 | 改为:`creditCard && (creditLimit != null \|\| billingDay != null \|\| dueDay != null)` 任一非空就显示 |
