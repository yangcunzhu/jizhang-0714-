## 🐛 实际遇到的问题

| 问题 | 解决 |
|---|---|
| Notifier 类型与 autoDispose provider 不匹配(`type_argument_not_matching_bounds`) | `RecordFormNotifier extends AutoDisposeNotifier<RecordFormState>` |
| 弹层 step 2 无"下一步"按钮,用户输入金额后卡住 | 加 `_PrimaryAction` 槽位,根据 step 显示"下一步"/"保存" |
| 保存按钮的 `CircularProgressIndicator` 是无限动画,pumpAndSettle 卡死 | 改成静态 `Text('保存中...')` |
| backspace 测试期望写错(12.30 → 实际是 12.03) | 修正:1234 cents backspace 一次 → 1203 cents = 12.03 元 |
| `ref.listen` 不会回调初始值,导致 accountId 永远为 null → canSubmit 永远 false → 保存按钮 disabled | 改 `ref.watch(defaultAccountProvider)` + `WidgetsBinding.addPostFrameCallback` 同步设置 accountId |
| `ref.listen` callback 中 `next.value` 是 dynamic,无法调 `.id` | 显式标注 `ref.listen<AsyncValue<AccountEntry?>>(...)` |
| 主页 FAB 接弹层时 SnackBar 占位 → 改成 `showRecordSheet(context)` 调用 | ✓ |

### 关键技术决策
1. **recordFormProvider 用 autoDispose**:弹层关闭后 Notifier 自动销毁,下次打开拿全新 state(避免上次半填状态污染)。Day 7 同样模式可复用。
2. **step 2 底部"下一步"按钮**:选分类自动跳 step 2 后,用户输入金额,需要明确"下一步"按钮推进。否则用户输入完金额无入口继续。`PrimaryAction` 槽位根据 step 显示"下一步"/"保存"。
3. **账户 ID 用 `ref.watch + addPostFrameCallback`**:不用 `ref.listen` 因为它不会回调初始值。在 build 中检测 `accountId == null && defaultAccountProvider.valueOrNull != null`,post-frame 调 setAccount。
4. **保存按钮不用 CircularProgressIndicator**:无限动画让 widget test 的 `pumpAndSettle` 卡死。改成静态 `Text('保存中...')`,功能等价 + 测试友好。
5. **振动 fire-and-forget 包装**:`Vibration.hasVibrator()` 在 widget test 抛 MissingPluginException,内层 try/catch 吞掉,不阻断 submit 返回。
6. **AccountPicker 用 StatefulWidget**:备注 TextField 的 controller 不能在 ConsumerWidget.build 中每次重建,否则光标跳末尾 + 输入丢失。
