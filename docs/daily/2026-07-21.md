### emoji vs Material Icons
- **emoji(推荐)**:跨平台一致(iOS 自动用 Apple Color Emoji)+ 用户直观 + 零字体依赖 + Stage 2 自定义分类支持 emoji 输入
- **Material Icons**:风格统一 + 不依赖系统字体 + Stage 2 自定义图标包(IconData)

**建议**:采用 emoji,理由:iPhone 用户熟悉 emoji,视觉冲击强,记账 App 常见做法(如 MoneyWiz、Money Lover)

## ⚠️ 用户注意

### 4. 关键技术决策
1. **`TransactionTile.formatYuan` 升为 public static**:不让 AccountPicker 自己重写一套 cents → 元 格式化函数,共享。
2. **emoji 字号强制 ≥ 20pt**:Flutter 文字 emoji 在小字号(< 16pt)渲染不清晰;CategoryGrid 24pt、TransactionTile 20pt、账户头像 18pt。
3. **"添加账户"独立按钮**而非下拉入口:Stage 1 单一账户明确显示,新账户用独立按钮提示(Stage 2 改为悬浮 FAB 或下拉菜单)。
4. **`_AccountCard` 拆为独立 Widget**:与 `_AccountPickerState` 解耦,便于 Stage 2 多账户时复用(`ListView.builder` 每行一个 _AccountCard)。
5. **保留 unused import 清理**:record_sheet.dart 之前 import `app_database.dart` 但没直接引用 — Day 7 一并删掉。
