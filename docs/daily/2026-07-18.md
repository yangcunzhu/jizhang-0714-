### 交付物

- 4 个 `*.g.dart` 生成物(需一并提交,见下)

### 关键设计决策
1. **金额用整数分(int cents)存储**,杜绝 double 浮点误差。
2. **支出/收入用 int 枚举 `TransactionType`**,金额恒正,方向由 type 区分。
3. **seed 默认数据放 MigrationStrategy.onCreate**,首次建库自动植入。
4. **加密延后 Stage 6**:Stage 1 用 NativeDatabase,schema 不变,届时换底层 executor。

### 依赖冲突处置(耗时主因)
- pub 初次把 Drift 工具链解析成坏组合:`drift_dev 2.34.0` 声明 `sqlparser ^0.44.0` 却调用 0.45.0 的 `.when` → build 崩。
- 根因链:`drift_dev ≥2.34.1` 需 `analyzer ^13`,而 `flutter_riverpod 3.x → riverpod 3.x → test` 锁 `analyzer <13`,互斥。
- **处置**:①`drift_dev` 钉到修复版 `^2.34.4`(拉起 sqlparser 0.45.0);②`flutter_riverpod` 降到 `^2.6.1`(**回归项目既定的 Riverpod 2.5+**,2.x 不拉 test);③移除 `riverpod_generator`/`riverpod_annotation`(Day 5 改手写 provider,更简更稳,无额外 codegen)。
- 结果:drift_dev 2.34.4 + sqlparser 0.45.0 + analyzer 13.0.0 + flutter_riverpod 2.6.1,全绿。

## 🔍 Day 4 独立审计 + 修复(督察角色)
