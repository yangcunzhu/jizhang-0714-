## 🆕 Day 15 关键实现

| 点 | 做法 | WHY |
|---|---|---|
| **模板元数据 vs 分类定义分离** | `category_templates` 表只存元数据(id/code/name/description/emoji);模板内分类用 Dart `const` `defaultTemplateDefinitions` 存 | 模板分类不入 categories 表,避免污染用户分类;应用时从 Dart const 读 → 去重 → 插入 |
| **applyTemplate 事务** | `transaction(() async { ... })` 包整段 | 中间失败自动回滚,数据一致 |
| **引用保护(覆盖模式)** | `countTransactionsByCategory` for each,> 0 保留 + < 0 删除 | 沿用 ADR-0019 §5 引用计数,不靠外键抛错 |
| **去重(name + iconName)** | `existingKeys` Set 缓存,插入前查 | 多次应用同一模板不重复 |
| **sortOrder 连续** | `nextSort = max(existing) + 1` + 递增 | 新分类排在已有之后,无重排 |
| **ApplyResult 数据类** | deleted / preserved / inserted / skippedDuplicate 4 计数 | UI toast 直接调 `summary()` 出一句话 |
| **主页 AppBar 入口** | `Icons.dashboard_customize_outlined` IconButton + `home-template-button` Key | 不破坏现有布局,易发现 |
| **空模板(自定义空)** | overwrite 模式仍走删除循环,append 模式空操作 | 用户能从零起步(overwrite)/ 纯空(append) |

## ⚠️ 用户注意

- **分类模板首次使用建议**:打开应用 → 主页右上角点「分类模板」图标 → 选「上班族」(12 个高频分类)→ 选「覆盖」→ 一键替换默认分类,开始记账
- **覆盖模式引用保护**:有交易的分类(如「餐饮」有 20 笔交易)不会被删除,会保留 + 跳过,toast 提示「已保留 N 个有引用分类」
- **模板应用是事务**:中间失败自动回滚,不会出现「删除一半新插一半」的脏数据
- **emoji picker 限制 72 个**:模板分类 emoji 全部从这 72 个精选里选;用户自建分类仍可任意 emoji
