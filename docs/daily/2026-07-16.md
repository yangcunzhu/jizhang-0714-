#### 详细步骤

**13:00-14:30 生成 CSR**

⚠️ **必须在 macOS 上生成**（GitHub Actions runner 自带 macOS，但不能交互）

方案 A：在 GitHub Actions runner 上生成（推荐）
- [ ] 创建临时 workflow `.github/workflows/_generate-cert.yml`（用 `workflow_dispatch` 手动触发）
- [ ] 用 `apple-actions/import-codesign-certs@v3` + `prometheuscyber/create-csr@v1` 组合
- [ ] （详细脚本见下方"高级方案"）

方案 B：在本机生成 CSR 文件 → 上传 Apple Developer
- [ ] **问题**：Windows 没有 Keychain Access，无法生成 macOS CSR
- [ ] **替代**：用 `openssl` 命令在 PowerShell 生成
  ```powershell
  # 生成私钥
  openssl genrsa -out ~/jizhang.key 2048
  
  # 生成 CSR
  openssl req -new -key ~/jizhang.key -out ~/jizhang.csr -subj "/CN=审计官 Development/O=YourName/C=CN"
  ```
- [ ] 用 `jizhang.csr` 上传 Apple Developer 创建证书

> **推荐方案 A**（GitHub Actions runner 生成）：更省事，不用本地存私钥

#### 详细步骤

**18:00-19:30 GitHub Secrets 配置**

- [ ] 浏览器打开 GitHub 仓库 → `Settings` → `Secrets and variables` → `Actions`
- [ ] 点击 `New repository secret`，添加以下 4 个：
  - `IOS_CERTIFICATE_BASE64`：粘贴 `cert_base64.txt` 内容
  - `IOS_PROVISIONING_PROFILE_BASE64`：粘贴 `profile_base64.txt` 内容
  - `KEY_PASSWORD`：CSR 私钥的密码（如果有）
  - `BUILD_CERTIFICATE_BASE64`：（如果用方案 A 自动生成）跳过
- [ ] 验证：所有 Secret 显示为绿色（值已加密）

**19:30-21:00 完善 workflow**

- [ ] 编辑 `.github/workflows/build-ios.yml`：
  ```yaml
  name: Build iOS .ipa
  
  on:
    push:
      branches: [ main ]
    workflow_dispatch:
  
  jobs:
    build:
      runs-on: macos-14
      timeout-minutes: 30
      
      steps:
        - name: Checkout
          uses: actions/checkout@v4
        
        - name: Setup Flutter
          uses: subosito/flutter-action@v2
          with:
            flutter-version: '3.24.5'
            channel: stable
        
        - name: Install dependencies
          run: flutter pub get
        
        - name: Import Apple certificate
          env:
            IOS_CERTIFICATE_BASE64: ${{ secrets.IOS_CERTIFICATE_BASE64 }}
            KEY_PASSWORD: ${{ secrets.KEY_PASSWORD }}
          run: |
            # 创建临时 keychain
            KEYCHAIN_PATH=$RUNNER_TEMP/build.keychain-db
            KEYCHAIN_PASSWORD=$(openssl rand -hex 16)
            
            security create-keychain -p "$KEYCHAIN_PASSWORD" $KEYCHAIN_PATH
            security set-keychain-settings -lut 21600 $KEYCHAIN_PATH
            security unlock-keychain -p "$KEYCHAIN_PASSWORD" $KEYCHAIN_PATH
            security list-keychains -d user -s $KEYCHAIN_PATH $(security list-keychains -d user | tr -d '"')
            
            # 导入证书
            CERT_PATH=$RUNNER_TEMP/cert.p12
            echo -n "$IOS_CERTIFICATE_BASE64" | base64 --decode > $CERT_PATH
            
            # 注意：需要先转换 .cer → .p12（包含私钥）
            # 实际方案：使用 apple-actions/import-codesign-certs 更简单
            # 见下方重构版
        
        - name: Import provisioning profile
          env:
            IOS_PROVISIONING_PROFILE_BASE64: ${{ secrets.IOS_PROVISIONING_PROFILE_BASE64 }}
          run: |
            PROFILE_PATH=$RUNNER_TEMP/profile.mobileprovision
            echo -n "$IOS_PROVISIONING_PROFILE_BASE64" | base64 --decode > $PROFILE_PATH
            mkdir -p ~/Library/MobileDevice/Provisioning\ Profiles
            cp $PROFILE_PATH ~/Library/MobileDevice/Provisioning\ Profiles/
        
        - name: Build .ipa
          run: |
            flutter build ios --release \
              --export-options-plist=ios/ExportOptions.plist
        
        - name: Upload artifact
          uses: actions/upload-artifact@v4
          with:
            name: jizhang-app-signed
            path: build/ios/iphoneos/Runner.ipa
            retention-days: 7
  ```
- [ ] 创建 `ios/ExportOptions.plist`：
  ```xml
  <?xml version="1.0" encoding="UTF-8"?>
  <!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
  <plist version="1.0">
  <dict>
      <key>method</key>
      <string>development</string>
      <key>provisioningProfiles</key>
      <dict>
          <key>com.shenjiguan.jizhang</key>
          <string>jizhang-dev</string>
      </dict>
      <key>signingStyle</key>
      <string>manual</string>
  </dict>
  </plist>
  ```
- [ ] 提交：
