### 决策记录
- **🟢 决策 #1**：放弃付费 Apple Developer Program（$99/年），采用 AltStore 0 成本路径
  - 理由：用户硬性约束"0 成本"（详见 product-design-v4.html §1 核心原则）
  - 后果：Bundle ID 由 Xcode Personal Team 自动生成，不可显式指定 `com.shenjiguan.jizhang`
  - 影响范围：S00 Day 3 部署 + 所有后续 Stage 的 Bundle ID 配置
- **🟢 决策 #2**：GitHub Actions 只产出未签名 .ipa，签名交给 AltStore
  - 理由：与产品文档 §2.12 §8.48 一致，简化 CI 配置
  - 后果：workflow 文件从 ~150 行（含 cert/profile）缩到 ~70 行
  - 影响范围：仅 S00；Stage 6+ 加密数据库签名仍需评估
- **🟢 决策 #3(2026-07-14 晚)**：部署架构两阶段化
  - 短期(开发期):Sideloadly + 开发机 USB + 手动 7 天续签
  - 长期(v1.0 上线后):AltServer + 旧电脑 WiFi 自动续签
  - 理由:AltServer 折腾几小时报错 + 旧电脑无 TYPE-C 不方便 + 开发期手动 30 秒可接受
  - Apple ID 不变(Personal Team 证书共用),切换方案 = 只换签名工具
  - 影响范围:S00 Day 3 + 整个 v1.0 开发周期
  - 详见 ADR-0004

### 今日学到的（Lessons）
1. **daily 文件不能凭直觉写**：原计划 10 步里 6 步是错的（基于"手动签名"的过时假设）。应先读 product-design-v4.html §2.12 部署架构，再写 daily
2. **fine-grained PAT ≠ classic PAT**：scope 限制更严，但 permission 可在 GitHub UI 单独加。已知坑：默认无 `actions:read`、无 `workflow` scope
3. **workflow artifact path 必须和 build 步骤实际产物路径一致**：用 `cd && zip` 时相对路径容易算错，解决方案：用绝对路径或 `cd` 后再 `ls` 验证
