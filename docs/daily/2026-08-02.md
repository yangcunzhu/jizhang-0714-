## 🆕 Day 19 关键实现

### 1. ADR-0022 余额自动更新策略(决策记录)

**位置**:`docs/adr/0022-balance-auto-update-strategy.md`(新,~150 行)

**4 项核心决策**:

| 决策 | 内容 | WHY |
|---|---|---|
| 1. `balanceCents` 统一语义 | 普通账户 = 当前余额;信用卡 = 已用额度 | 单字段不分裂,还款流逻辑简单 |
| 2. `_updateAccountBalance` silent skip v2 | v1 抛错 → v2 静默更新(允许透支)| S02 测试 fixture 默认 0 余额,v1 抛错破坏 31 个测试 |
| 3. `transferRepayment` 显式 check | 储蓄余额不足抛 StateError + 事务回滚 | 还款是主动行为,余额校验必要 |
| 4. `_getOrCreateRepaymentCategoryId` 自动 seed | 首次还款时创建「还款」分类,后续复用 | 不用手动 seed 默认分类(S02 写集已收尾)|

**不可逆性**(ADR-0022 §不可逆性):
- `balanceCents` 语义(普通 = 余额,信用卡 = 已用)— 永不变更
- 还款 transaction 引用「还款」分类 — 永不变更
- 还款语义「储蓄 -amount + 信用卡 -amount」— 永不变更

### 2. transaction_dao.dart 4 个方法

**`_updateAccountBalance(accountId, deltaCents)`**(私有):
- silent skip on 账户不存在(测试 fixture / 已删账户)
- 允许透支(余额可负,信用卡已用可负)
- 余额校验 **不在此处**(业务方法显式 check)

**`transferRepayment(fromSavingsAccountId, toCreditCardAccountId, amountCents, note?)`**(公开):
- `transaction {}` 包裹,原子化
- 校验 6 维度:amount > 0 / 储蓄存在 / 信用卡存在 / 信用卡类型正确 / 储蓄余额足够
- 更新两个账户余额 + 写 repayment transaction + 引用「还款」分类
- 任意校验失败 → 抛 StateError/ArgumentError,事务回滚

**`_getOrCreateRepaymentCategoryId()`**(私有):
- 查找 name='还款' 的分类,存在则返回 ID
- 不存在则创建(name='还款', icon='💳', color=紫色 0xFF7E57C2, type=expense, sortOrder=10)

**`insertTransaction(entry)` 联动改造**:
- 支出:`_updateAccountBalance(accountId, -amountCents)`
- 收入:`_updateAccountBalance(accountId, +amountCents)`
- repayment:抛 `ArgumentError`(必须走 transferRepayment 双账户事务)
