## 🆕 Day 11 关键决策(开工会话对齐)

### AccountType enum 取值

候选:
- **方案 A — 中文 enum**:`cash / savings / credit / huabei / loan / investment`
- **方案 B — 英文 enum**:`cash / savings / creditCard / huabei / onlineLoan / investment`
- **方案 C — 双语**(值 = 英文,显示 = 中文)

**推荐 C**:enum 存英文(数据库 i18n 安全),UI 显示中文 + 类型 emoji。

### Schema migration 策略

- **方案 A — 增量 ALTER TABLE**(Drift 标准 migration API)
- **方案 B — 删表重建**(破坏 Stage 1 数据,排除)
- **方案 C — 新表平迁**(复杂,排除)

**推荐 A**:5 个新字段 + 默认值兜底,Stage 1 单"现金"账户自动归类 cash。

### 关键决策

1. **AccountType 6 值走 ADR-0017 推荐 C**:`textEnum<AccountType>()` 存英文 name,UI 用 `displayName`(中文)+ `emoji` getter。
2. **Schema migration 走 ADR-0017 推荐 A**:`onUpgrade(from, to)` 用 `m.addColumn` 增量 ALTER 5 列,Stage 1 数据 0 丢失。
3. **新字段可空性**:`type`(NOT NULL DEFAULT 'cash') / `includeInNetWorth`(NOT NULL DEFAULT true) — 兜底 Stage 1 已有账户自动归类;`creditLimit` / `billingDay` / `dueDay`(nullable)— 非信用卡账户 NULL 是语义正确。
4. **migration 测试用 file-based NativeDatabase**:`NativeDatabase.memory()` close 后不能 reopen,改用 `tempDir` + `File(path)` 模拟用户真机升级路径。

### 调试过程(沉淀)

- **错误 1**:`AccountEntriesCompanion` 不存在。Drift 默认 companion 命名是表名 + `Companion`,不是 `DataClassName` + `s` + `Companion`。`Accounts` → `AccountsCompanion`。
- **错误 2**:`update(..write(entry)).go()` 链用法错。Drift 标准:`(update(table)..where(...)).write(entry)` 即可,`.write()` 返回 `UpdateStatement` 已 bind 到 connection。
- **错误 3**:migration 测试用 `NativeDatabase.memory()` close 后不能 reopen(`Bad state: Can't re-open a database after closing it`)。改用 file-based:`NativeDatabase(File(tmpPath))`,close 后文件保留,再开同路径触发 onUpgrade。
- **错误 4**:`isNull`/`isNotNull` 命名冲突 — `package:drift/drift.dart` 和 `package:flutter_test/flutter_test.dart` 都导出。`import drift.dart hide isNull, isNotNull`(沿用 `app_database_test.dart` 的写法)。
- **错误 5**:`customStatement` 不是 `NativeDatabase` 的方法,是 `Database` 上的。测试改用 `AppDatabase.forTesting(rawDb)` → `db.customStatement(...)`。
