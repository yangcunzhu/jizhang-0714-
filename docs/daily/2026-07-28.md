## 🆕 Day 14 关键实现

| 点 | 做法 | WHY |
|---|---|---|
| **iconName 语义澄清** | categories.dart 注释从「Material/Cupertino codepoint」改为「emoji 字符 (UTF-16)」 | ADR-0019 §1:消除误解,后续维护者不会尝试 codepoint 解析 |
| **category_dao 扩表** | `@DriftAccessor(tables: [Categories, Transactions])` + countTransactionsByCategory | 引用计数查同 DAO,不跨 DAO(避免越界) |
| **swapSortOrder 原子交换** | 两行 UPDATE 互交换,无需事务 | 单步移动语义,UI 与 DAO 一致 |
| **category_form_provider** | StateNotifier + family `({int? existingId, TransactionType type})` record key | 新建支出/收入同时打开不串台 |
| **category_form_state.sortOrder** | 新建场景 controller 异步取 max+1 | 无需重排已有分类 |
| **emoji_picker** | 6 主题 × 12 = 72 emoji + 静态 `kEmojiGroups` const 数据 | 离线、零依赖、可测 |
| **12 色板** | Wrap 展示,选中加 onSurface 边框 + check icon | 不引 color_picker 包,避免 §2 新依赖 |
| **category_edit_sheet** | 弹层 + 实时预览(emoji+色+名)+ emoji picker 按钮 | 与 account_edit_sheet 同模式,风格统一 |
| **category_management_page** | ListView.separated + 每行 ↑↓ 按钮 + 引用数小标 + 长按弹菜单(编辑/删除) | 与 account_management_page 同模式 |
| **删除有引用禁用** | `categoryReferenceCountProvider` + delete ListTile `enabled: false` + tooltip 文案 | 用户立刻知道"为什么不能删",不靠外键抛错 |
| **category_grid 长按回调** | 新增 `onManageCategory: VoidCallback?` + 「+新增」tile | 长按或点 + → 跳管理页(record_sheet 接线) |

## 🐛 遇到问题与解决

| 问题 | 解决 |
|---|---|
| category_grid_test 旧 hostWithGrid 缺 onManageCategory 参数 | hostWithGrid 改为 optional 参数,向后兼容 |
| 12 色板 hex 字符串写错("ffffe57373" 多 2 位) | `0xFFE57373.toRadixString(16)` = "ffe57373" → 修正 |
| ListView 懒构建:末尾 item 不可见 → find.byKey 空 | 关键 finder 加 `skipOffstage: false` |
| Wrap 内 12 色板 off-screen → 同上 | `skipOffstage: false` |
| categoryFormProvider family key 类型 doc 用 `<>` 被 lint 当 HTML | doc 改 backticks |
| insert 传 `type: Value(state.type)` 类型不匹配 | insert 用 enum 值,不需 Value 包装(textEnum 自动包装) |
| record_sheet 收尾卡显示 `use_build_context_synchronously` warning risk | 加 `// ignore: use_build_context_synchronously` 注释(ModalSheet 已挂载,context 安全) |
