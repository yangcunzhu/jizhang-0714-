### 2. Schema migration v3 → v4(核心改动)

**WHY**:ADR-0021 决定用 `textEnum` 模式扩展 TransactionType 加 `repayment` 值(S03 还款流)。textEnum 加 enum 值时 SQLite 列定义不变(仍 TEXT),仅 Dart 层 enum 多一个常量,**零迁移成本**。

**改动文件**:
- `lib/data/db/app_database.dart` — schemaVersion 3 → 4 + onUpgrade 加 `if (from < 4) { /* 占位 */ }` + 类头注释加 v4 行
- `lib/data/db/tables/categories.dart` — TransactionType enum 加 `repayment` + L4-7 注释从 intEnum 改 textEnum + 不可逆性说明
- `lib/data/db/tables/transactions.dart` — L18 注释「支出 / 收入」→「支出 / 收入 / 还款」+ 加 textEnum 兼容性说明

**不可逆性**(ADR-0021 §「不可逆性」):
- `repayment` 名称必须永不变更,下游月度还款总额统计依赖字符串匹配
- 0 新依赖(不引 flutter_local_notifications,沿用现有栈)
