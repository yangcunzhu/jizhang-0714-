## 🆕 实施核心要点

### 1. C-IQA-D28-1:`_monthlyStatsProvider` 自动 invalidate

**修改**:`lib/features/home/presentation/home_page.dart`

```dart
final _monthlyStatsProvider = FutureProvider.autoDispose<MonthlyStatsSnapshot>(
  (ref) async {
    // IQA-fix:建立 StreamProvider 依赖,自动 invalidate on transaction change。
    ref.watch(transactionListProvider);
    final db = ref.read(databaseProvider);
    return db.statisticsDao.getMonthlyStats(DateTime.now());
  },
);
```

**WHY work**:`ref.watch(transactionListProvider)` 在 FutureProvider 内 — Riverpod 自动建立依赖。StreamProvider emit 新值时,FutureProvider 自动 invalid + 重新 fetch。

**failure scenario 修复**:
- 旧:用户开 toggle「不计收支」提交,主页 income 不刷新
- 新:用户开 toggle 提交 → `ref.watch(transactionListProvider)` 触发 Stream emit → invalidate `_monthlyStatsProvider` → 主页 rebuild 显示 ¥0.00 变化

### 2. G-IQA-D28-6:refund 编辑 toggle 锁

**修改**:
- `lib/features/record/application/record_form_provider.dart`:
  - `RecordFormState.isRefundLocked = false` 新字段
  - `loadForEdit` 检测 `tx.type == TransactionType.refund` 时 → `isRefundLocked = true`
  - `setExcludeFromIncomeExpense/SetExcludeFromBudget`:若 `isRefundLocked` 早 return(用户改不动)
  - `submit()` 编辑模式:若 `old.type == refund` → `excludeFromIncomeExpense`/`excludeFromBudget` pass 老值(不写)— 保证 toggle=true 不变

- `lib/features/record/presentation/record_sheet.dart`:
  - SwitchListTile `onChanged`:若 `form.isRefundLocked` → `null` chip disabled
  - title 文本:`'不计收支(锁定)'` vs `'不计收支'`(refund 编辑时显式标注)
