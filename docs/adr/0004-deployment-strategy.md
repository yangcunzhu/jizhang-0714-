# ADR-0004: 部署架构两阶段化(Sideloadly 短期 + AltServer 长期)

> **状态**: 已接受
> **日期**: 2026-07-14
> **决策者**: 用户(指挥官) + Claude(大副执行)
> **ADR 系列**: 0001 Flutter 技术栈 · 0002 项目结构 · 0003 测试策略

---

## 背景

Stage 0 环境验证阶段,需要把 .ipa 装到 iPhone 真机。最初按 `product-design-v4.html §2.12 部署架构 + §8.48 AltStore 部署` 规划:

> 家里旧电脑装 AltServer(7×24 在线),iPhone 通过 WiFi 自动续签

实施时发现实际场景与文档假设不完全一致:

| 维度 | 文档假设 | 实际场景 |
|---|---|---|
| AltServer 实施难度 | 文档描述"运行 AltServer 即可" | 实操需要 Apple Mobile Device Service + AltStore App 配对 + iPhone 信任证书,几小时折腾仍报错 |
| 旧电脑接口 | 未明确 | 旧电脑无 USB-C,iPhone 15/16 用 USB-C,**首次配对物理连接不便** |
| 用户使用模式 | "App 7×24 自动续签" | 用户每天 7:00-24:00 旧电脑开机(覆盖使用时间) |
| 开发节奏 | 长期稳定部署 | 当前是 S00 v1.0 开发期,**开发机更顺手** |

## 决策

**部署架构分两阶段**:

### 短期(2026-07 → v1.0 上线前 · 开发期)

- **工具**: Sideloadly(独立 GUI,免配对)
- **设备**: 开发机 + iPhone USB-C 连接
- **续签**: 每 7 天手动一次(30 秒)
- **优点**: 简单直接,无 AltStore 中转复杂度
- **缺点**: 必须每 7 天记得手动续签

### 长期(2026 v1.0 上线后 · 正式使用)

- **工具**: AltServer(后台) + AltStore(App 中转)
- **设备**: 旧电脑(7×24 在线) + iPhone(同 WiFi 自动续签)
- **续签**: 自动(条件:旧电脑在线 + iPhone 充电 + 同 WiFi)
- **优点**: 零维护
- **缺点**: 首次配对 30 分钟

## 关键不变量

| 项 | 说明 |
|---|---|
| **Apple ID** | **不变** — Personal Team 免费证书两方案共用 |
| **Bundle ID** | 短期 Sideloadly 可自定义,长期 AltServer 强制 Xcode 自动生成 |
| **Runner.ipa** | 不变(同一产物) |
| **切换方案** | 不需要重装 App 数据,不需要换 Apple ID,只换签名工具 |

## 后果

### 正面
- ✅ 开发期 Sideloadly 简化 Day 3 真机验证流程(避免 AltServer 几小时报错)
- ✅ 长期 AltServer 提供零维护体验,适合 v1.0 上线后
- ✅ Apple ID 不变,降低用户操作复杂度
- ✅ 充分利用用户已有资源(开发机 + 旧电脑)

### 负面 / 风险
- ⚠️ 开发期需要每 7 天手动续签(忘记则 App 打不开)
  - 缓解:在 ROA 文档 + Daily 提醒
- ⚠️ 切换长期方案时需要 30 分钟一次性配对
  - 缓解:写入 Stage 1+ 的标准操作流程

### 中性
- 🟦 旧电脑短期内不强制要求(仅作为"未来切换"的资源)
- 🟦 旧电脑 Claude(旧 Claude 实例)退役(短期方案不需要)

## 切换触发条件(2026 年 v1.0 上线时评估)

- [ ] v1.0 上线
- [ ] 用户希望"零维护"(不愿每 7 天手动)
- [ ] 旧电脑 7:00-24:00 稳定开机(已具备)
- [ ] Apple ID 仍可用

满足以上条件 → 切 AltServer 路径。

## 替代方案(已考虑并放弃)

### A. 一开始就 AltServer
- ❌ AltServer 配对几小时报错,影响 Day 3 进度
- ❌ 旧电脑无 USB-C,首次配对不便
- ✅ 零维护

### B. 一开始就 Sideloadly,长期不切
- ✅ 最简单
- ❌ 长期每 7 天手动续签(用户意愿未知)
- 待评估:用户用 v1.0 后是否愿意手动

### C. TestFlight(付费 Developer Program)
- ❌ $99/年,违反 0 成本原则
- ✅ 自动续签 + 多设备支持
- 排除

### D. App Store(免费)
- ❌ 审核周期 + 上架要求复杂
- ❌ 自用 App 不适合上架
- 排除

## 备注

本 ADR 的关键洞察:**部署策略应根据开发阶段动态调整**,而不是一开始就追求"完美长期方案"。开发期 = 简单直接,正式期 = 零维护。两者切换成本低(Apple ID 不变)。
