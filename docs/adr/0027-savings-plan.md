## §2 决策

### 1. 攒钱计划 5 模式(产品形态)

| 模式 | 规则 | 适合 |
|---|---|---|
| 365 天存钱 | 第 N 天存 N 元(1+2+3+...+365 = 66795 元)| 想养成每日习惯 |
| 52 周存钱 | 第 N 周存 N × 10 元(10+20+...+520 = 13780 元)| 喜欢周节奏 |
| 12 月存钱 | 第 N 月存 N × 100 元(100+200+...+1200 = 7800 元)| 稳定月收入 |
| 定额存钱 | 每月固定存 X 元(用户设)| 严格预算 |
| 灵活存钱 | 任意金额任意时间 | 自由 |

**关键洞察**:
- 这是**第三方资金管理**,不是 App 自管
- 咔皮借支付宝小荷包的能力,App 不碰用户的钱
- 用户充值→小荷包冻结→到期可取(类似余额宝)
- App 责任是「记账 / 提醒 / 心理反馈」,不是「资金安全」

### 2. 与支付宝小荷包合作(关键)

- 咔皮**不碰用户钱**,只做记账 + 提醒
- 钱实际在小荷包(支付宝体系),安全保障 = 支付宝背书
- 充值 / 取现 / 消费 → 走支付宝流程
- App 只记录:「用户 7 月 18 日开始 365 天存钱计划,第 1 天存了 1 元」

**WHY 重要**:
- 我们(《审计官》)**不需要实现**完整金融功能
- 集成第三方 API(支付宝小荷包)即可,但这超出 v1.0 范围
- v1.0 阶段:**「存钱计划」仅是记账记录**,不接支付宝 API
- v1.1+:评估接支付宝小荷包(需要商务合作)

### 3. 桌面小组件(图 16 / 18 / 19)

- **iOS 14+ 系统要求**(图 16 FAQ)
- 桌面小组件显示:
  - 今日消费(¥128.7 万)
  - 最近 4 笔交易记录(12:30 餐饮 / 10:15 交通 / 09:45 购物 / 08:20 娱乐)
  - 总账本汇总数据
- **数据源限制**:只显示「总账本」,分账本暂不支持
- **HOW 添加**:返回主屏 → 桌面空白处长按 → 编辑模式 → 添加组件 → 搜索「咔皮记账」→ 添加

**系统要求**:
- iOS 14+ WidgetKit
- 小组件代码量较大(独立 Widget Extension target)
- v1.0 范围:**不做**小组件(v1.1 评估)
- 现阶段只做 App 内 UI

### 4. 攒攒 IP 智能反馈规则(图 1 / 15)

- **IP 形象**:胖咔(小动物,像仓鼠 / 啮齿类)
- **反馈文案**根据不同时段:
  - 图 1(中午 21:07):「今日消费恰好等于预算,你的规划力超稳」 ✅
  - 图 15(晚上 22:34):「**今日预算使用率 100%,为理性的你点赞**」 ✅
- **可能规则**(待用户实际触发更多场景):
  - 0-30% → 攒攒鼓励
  - 30-80% → 进度提醒
  - 80-100% → 节流肯定
  - 100%+ → 警告 + 解释「理性消费」

**完整规则**需要在咔皮多次触发不同状态确认,**等待用户补充**。

## §3 不可逆性

| 项 | 永不变更 | 理由 |
|---|---|---|
| 攒钱计划 5 模式命名 | 不可变 | 用户已习惯,改了有迁移成本 |
| 资金第三方(支付宝小荷包)合作 | 不可变(如果 v1.1 接) | 支付宝背书,资金安全是核心 |
| iOS 14+ 桌面小组件系统要求 | 不可变 | 平台限制 |
| 总账本 vs 分账本 数据源 | 不可变 | 性能 + 简化 |
