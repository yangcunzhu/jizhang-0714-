> 状态:**DRAFT — 待用户拍板**

> Stage:**S02 补救 + S03 完善**
> 作者:Claude(执行)+ 用户(决策)
> 关联:ADR-0017(账户 schema v2)+ ADR-0018(账户 UI)+ ADR-0021(S03 范围)+ CLAUDE.md 铁律 7/8

---

## 背景

D18 用户真机手验,发现 S02 阶段账户管理存在**严重设计缺陷**:

1. **「初始余额」字段漏掉** — 用户没法设现金有 ¥5000
2. **6 种账户类型产品逻辑一刀切** — 花呗、网贷、理财跟现金用同一逻辑
3. **还款流只支持「储蓄 → 信用卡」** — 不支持理财赎回还信用卡、花呗、网贷
4. **自动还款没做** — 用户期望 A+ 级产品该有的
5. **还款提醒降级** — ADR-0021 改为卡片显示「距离还款日 X 天」,但根本原因是省事(不引 flutter_local_notifications)
6. **理财概念模糊** — S02 阶段说「理财」包括基金/股票/P2P,产品定位不清

**根因**:CLAUDE.md §1 「极致体验 / 顶级品质 / 行家认可」没主动遵守,S02 阶段偷懒,S03 阶段用「MVP 简化」掩盖了产品逻辑漏洞。

---

## 决策

### 1. 6 种账户类型明确定位(产品差异化)

| 类型 | 产品定位 | 用户视角 | 还款策略 |
|---|---|---|---|
| 💵 **现金** | 物理现金钱包 | 「我钱包里有 ¥500」 | 无(直接用)|
| 🏦 **储蓄** | 银行存款账户 | 「我银行卡有 ¥5000」 | 无(直接用)|
| 💳 **信用卡** | 信用卡(招行 / 兴业 / 中信等)| 「我欠招行 ¥-500」 | 还款:储蓄 → 信用卡(已实现)|
| 🅰️ **花呗** | 蚂蚁花呗 | 「我欠花呗 ¥-300」 | 还款:储蓄 → 花呗(待实现)|
| 🆘 **网贷** | 网贷(借呗 / 微粒贷 / 360 等)| 「我欠借呗 ¥-1000」 | 还款:储蓄 → 网贷(待实现)+ **期数管理**(待实现)|
| 📈 **理财** | **明确为「公募基金」**(股票/P2P v2.0)| 「我基金账户 ¥10000」 | 赎回 → 信用卡/储蓄(待实现)|

**WHY 明确理财 = 公募基金**:
- 基金有「净值」「持仓」概念,记账软件(如 Money Pro)也这么归类
- 股票需要单独行情模块(v2.0 再做)
- P2P 大部分已暴雷,不归类

### 2. 「初始余额」字段已加(D19 后期 commit `4f5954e`)

所有 6 种账户类型都有「初始余额」输入框,新建时填、编辑时可调。

### 3. 还款流产品差异化(本次必补)

**当前**:只有 `transferRepayment(fromSavingsAccountId, toCreditCardAccountId, ...)`(S03 D19)
**目标**:统一所有「欠款类账户」的还款接口

```dart
// 重构后(本次任务):
Future<int> transferRepayment({
  required int fromAccountId,        // 储蓄 / 现金 / 理财赎回
  required int toAccountId,          // 信用卡 / 花呗 / 网贷
  required int amountCents,
  String? note,
  int? installmentPeriod,            // 网贷专属(可选,如「12期」)
});
```

**3 类场景**:

| 场景 | 储蓄(扣) | 信用卡(收) | 备注 |
|---|---|---|---|
| **正常** | ¥5000 → ¥4500 | 已用 -500 | ✓ |
| **异常(储蓄不够)**| 余额 ¥200,还款 ¥500 | 不变 | 抛 `StateError(余额不足)`,事务回滚 |
| **边界(amount=0)**| 不变 | 不变 | 抛 `ArgumentError`,submit 早返回 |

**网贷期数**(新功能):
- 字段:`installmentPeriod int?(可空)`(12 期 / 24 期 / 36 期)
- UI:还款弹层加「期数」下拉(仅网贷类型显示)
- DB:transactions 表加 `installment_period INTEGER` 列(schema v5)

### 4. 自动还款(本次补 ADR,本次不实现)

**WHY 推迟到 v1.1**:
- 自动还款需要后台定时任务(本地 notification + workmanager)
- flutter_local_notifications + workmanager 都是新依赖
- CLAUDE.md §2「不引入新依赖」(S03 0 新依赖)
- **本期不做**,但 ADR 记录「什么时候做」

**v1.1 计划**:
- 还款日前 3 天发通知(本地)
- 用户在 App 内一键触发自动还款
- 后端任务调度(本地 workmanager)

### 5. 还款提醒(D19 后期降级 → 本次评估)

**当前**(ADR-0021):**不引 flutter_local_notifications**,卡片显示「距离还款日 X 天」
**评估**:这是个**真正降级**。A+ 产品应该本地通知 + 卡片显示双通道。

**本次补 ADR-0025**(S01 backlog),v1.1 实现通知:
- 本地通知(flutter_local_notifications)
- 触发:还款日前 3 天 / 1 天 / 当天,4 个时间点
- App 内「设置 → 通知」开关(用户控制)

**本次不实现**,但 ADR 记录。

### 6. 理财产品特殊处理

**v1.0**:仅作为「可还款的资产账户」(可用余额赎回还信用卡/储蓄)
**v1.1**:加「持仓」「净值」概念,需要实时行情(API 调用)

**本期**:理财账户的还款流与信用卡相同(transferRepayment),不特殊处理。
**v1.1**:理财赎回单独流程(API + 持仓查询)。

## 不可逆性

| 项 | 永不变更 | 理由 |
|---|---|---|
| 6 种账户类型的产品定位 | 不可变更 | 用户已经习惯,改成本极高 |
| 还款 transaction type=repayment 语义 | 不可变更 | 下游统计依赖 |
| 信用卡 `balanceCents` = 已用额度 | 不可变更 | S05 净资产计算依赖 |
| 网贷 `installment_period` schema | 不可变更(本次加)| 记账流水统计依赖 |

## 后果

### 正面影响

- ✅ 花呗 / 网贷 / 理财**真正支持还款**,不再是「假支持」
- ✅ 网贷分期管理(v1.1 完善,本期加字段)
- ✅ 错误处理覆盖率:正常 / 异常 / 边界 3 类全覆盖

- ✅ CLAUDE.md 铁律 7/8 加,未来 Stage 不再偷懒

### 负面影响 / 风险

| 风险 | 等级 | 缓解 |
|---|---|---|
| 还款流接口重构(transferRepayment 参数) | 🟡 中 | S03 D19 测试用例已覆盖储蓄→信用卡,扩展后回归测试 |
