# UX 规范

> 状态：`GOVERNANCE`
> 创建：2026-07-14
> 适用范围：所有交互设计

---

## 🎯 核心 UX 原则

1. **3 步以内**：每个核心操作不超过 3 步
2. **零歧义**：文案清晰、明确，不让用户猜
3. **可逆**：危险操作有确认 + 可撤销
4. **响应即时**：所有操作 < 100ms 视觉反馈
5. **错误友好**：错误信息告诉用户**怎么办**，不只说哪里错了

---

## 📱 5 个核心交互

| 场景 | 步骤数 | 完成时间 |
|---|---|---|
| 记一笔 | 3 步 | ≤ 5 秒 |
| 设置预算 | 3 步 | ≤ 30 秒 |
| 还款 | 3 步 | ≤ 30 秒 |
| 切换账本 | 1 步 | ≤ 1 秒 |
| 查看统计 | 2 步 | ≤ 3 秒 |

```

   ↑                                            ↓
   └──────── 列表自动更新 ←─────────────────────┘
```

## ✍️ 文案规范

### 动词开头，避免名词

```dart
// ✅ 好
'保存'  '删除'  '添加分类'  '切换账本'

// ❌ 差
'保存按钮'  '删除操作'  '分类的添加'
```

### 简短

```
✅ 添加    ❌ 点击此处添加新分类
✅ 删除    ❌ 是否要执行删除操作？
✅ 取消    ❌ 放弃当前编辑并返回
```

### 主动语态

```
✅ 已保存    ❌ 你的修改已被保存
✅ 删除成功  ❌ 数据已被删除
```

### 错误信息：告诉用户怎么办

```dart
// ✅ 好
'金额必须大于 0，请重新输入'
'网络断开，请检查 WiFi 或使用离线模式'

// ❌ 差
'错误：输入无效'
'Network Error'
```

## 🔘 按钮位置

### 主要操作

- **右下角**（拇指自然落点）
- 例：主页"记一笔"按钮、记账卡"保存"

### 次要操作

- **左上角**（返回 / 取消）
- 或弹层底部居中

### 危险操作

- **红色文字按钮**（左下）
- 例：删除、清空

```
┌─────────────────────────────┐
│ [取消]              [删除] │  ← 危险操作红色
└─────────────────────────────┘
```

## ⚡ 即时反馈

### 视觉反馈

| 操作 | 反馈 |
|---|---|
| 点击按钮 | 颜色加深 + 100ms |
| 保存成功 | Toast + 振动 + 攒攒动画 |
| 网络请求 | Loading 指示器 |
| 错误 | 抖动 + 红色 Toast |

### 振动反馈

- 成功：轻振动（`HapticFeedback.lightImpact`）
- 警告：中振动（`HapticFeedback.mediumImpact`）
- 错误：重振动（`HapticFeedback.heavyImpact`）

### 攒攒动画

- 记账成功：攒攒弹跳 + 撒花
- 攒钱成功：攒攒旋转 + ✨
- 超支：攒攒摇头 + 😢
- 还款：攒攒点头 + 💪

## 📜 列表设计

### 列表项

- 高度：56-72pt
- 左边：图标/头像（48×48）
- 中间：标题 + 副标题
- 右边：金额 + 箭头

```
┌────────────────────────────────────┐
│ 🍔  餐饮                  -¥50.00 ›│
│      今天 12:30 · 现金              │
└────────────────────────────────────┘
```

### 分组

- 按日期分组（今天、昨天、本周、本月、更早）
- 每天小计显示

### 滚动

- 上拉加载更多（pagination）
- 下拉刷新（pull-to-refresh）
- 长按显示快捷操作（修改、删除、退款）

## 📝 表单设计

### 输入框

- **Label 在上**（Material 风格）而非内嵌（iOS 风格），因为更适合中文
- 占位符用 hint 而非 label
- 错误信息在输入框下方

```
┌──────────────────────┐
│ 金额                  │  ← Label
│ ┌──────────────────┐ │
│ │ ¥ 0.00           │ │  ← Input
│ └──────────────────┘ │
│ ⚠ 金额必须大于 0     │  ← Error (optional)
└──────────────────────┘
```

### 键盘

- 金额：`TextInputType.number`（数字键盘）
- 文本：`TextInputType.text`
- 多行：`TextInputType.multiline`
- 自动 focus：进入页面后第一个输入框自动 focus

## 🎯 空状态（Empty State）

任何列表为空时，显示引导：

```
┌─────────────────────────────┐
│                             │
│         📝                  │
│                             │
│     还没有记账记录           │
│                             │
│  点击下方按钮记一笔吧         │
│                             │
│    [+ 记一笔]                │
│                             │
└─────────────────────────────┘
```

**禁止**：纯空白让用户不知所措

## ⚠️ 加载与错误

### 加载

- 全屏加载：用于首次加载
- 局部加载：用于分块加载（骨架屏）
- 按钮加载：按钮内显示 spinner，按钮禁用

### 错误

| 类型 | 表现 |
|---|---|
| 网络断开 | Toast + 离线模式提示 |
| 服务器错误 | Toast + 重试按钮 |
| 数据错误 | 内联错误（输入框下方） |
| 权限错误 | 全屏提示 + 去设置 |

---

## 🔄 撤销

危险操作必须可撤销：

```
删除交易 → Toast 显示"已删除" + 撤销按钮（5 秒）
```

撤销期间不真正删除，过期后才删除。

## 📱 iOS HIG 合规

### 必须遵守

- [x] 状态栏：不要覆盖，除非必要
- [x] Home Indicator：底部 34pt 安全区
- [x] 返回手势：从左边缘右滑
- [x] 触控目标 ≥ 44pt
- [x] 系统字体（Dynamic Type 支持）

### 禁止

- [ ] 不要做 iOS 风格的自定义返回按钮（用系统返回手势）
- [ ] 不要做 Android 风格的 Material 按钮（除非必要）
- [ ] 不要覆盖系统手势
- [ ] 不要忽略 SafeArea

## 🌙 深色模式

v1.0 仅浅色，但预留 token（见 design-system.md）。

v1.1 启用时，遵守：

- 文本对比度 ≥ 4.5:1（WCAG AA）
- 避免纯黑（用 #0F172A）
- 避免纯白（用 #F1F5F9）

---

## ♿ 可访问性（a11y）

### 必须

- 所有交互元素有 `Semantics` label
- 颜色不是唯一信息载体（加图标或文字）
- 支持 Dynamic Type（用户字号偏好）
- 触控目标 ≥ 44pt

### 推荐

- 减少动画（`prefers-reduced-motion`）
- 高对比度模式

---

## 📌 UX 检查清单（每个 UI Task 完工）

```
□ 主操作 ≤ 3 步
□ 主按钮在右下（拇指可达）
□ 文案动词开头、简短
□ 错误信息告诉用户怎么办
□ 危险操作有确认 + 撤销
□ 即时反馈（视觉/振动）
□ 列表有空状态引导
□ 表单有 label + error 提示
□ iOS HIG 合规（SafeArea、触控目标）
□ Semantics 标签
□ 设计 token 使用（颜色/间距/字体）
```
