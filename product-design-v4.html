§3.1 2026-08-03 补充：基于 31 张咔皮截图的真实产品对标

ADR 引用:ADR-0026 13 张图完整分析 / ADR-0027 攒钱计划分册。

账户子类型(真实品牌清单,共 17+ 子类)

  • 资金账户(资产,5):储蓄卡 / 微信 / 支付宝 / 现金 / 自定义
  • 信用账户(负债,7):信用卡 / 花呗 / 京东白条 / 借呗 / 自定义 + 抖音放心贷 + 抖音月付账户(我之前漏 3 个)
  • 充值账户(资产,3):公交卡 / 饭卡 / 自自定义
  • 理财账户(资产,6):股票 / 基金 / 余额宝 / 零钱通 / 定期存款 / 自定义
  • 借贷账户(独立账户,2 子类):借出(资产,subType=lend) / 借入(负债,subType=borrow)

⚠️ v4 §3 原计划「4 大类 × 17 子类」被 ADR-0026 §10 修正为「5 大类 × 17+ 子类」(借贷作为独立账户,不是 transaction 类型)。

信用账户 6 字段(ADR-0026 §11 修正)

v4 §3 之前只说「信用卡 3 字段(额度/账单日/还款日)」。实际咔皮是 6 字段:信用额度 / 起始欠款 / 可用额度(自动算) / 起始时间 / 出账日 / 还款日。「起始欠款」+「起始时间」+「可用额度(自动算)」是关键新字段。

借贷 6 字段(ADR-0026 §12 修正)

借贷是独立账户(不是 v4 之前说的「记录」)。字段:起始余额(借出)/起始欠款(借入)+ 起始时间 + 账户名称 + 备注 + 借款人姓名 + 借出/借入日期 + 还款日期 + 扣款/入款账户 + 3 toggle。

记账弹层 3 tab(ADR-0026 §4)

v4 之前只说「支出/收入」。实际是支出/收入/转账 3 tab。转账是独立流程(扣款→入款,双方都是普通账户)。

转账 vs 还款 区别(ADR-0026 §5)

  • 转账:扣款 → 入款(双方都是普通账户资金类)
  • 还款:扣款(储蓄)→ 入款(欠款类)

桌面小组件 + 攒钱计划(ADR-0027 单独分册)

v4 §3 没覆盖。新模块:攒钱计划 5 模式(365天/52周/12月/定额/灵活) + 与支付宝小荷包合作 + iOS 14+ 桌面小组件 + 攒攒 IP 智能反馈。详见 ADR-0027。v1.0 只做「记账模式」,实际资金扣划 v1.1 接支付宝 API。

账本维度(ADR-0026 §12.4)

v4 之前没提账本。实际是必填字段:多账本切换(家庭/工作/个人),右上角切换。所有 transaction 查询带 WHERE ledgerId filter。分账本暂不支持桌面小组件数据。

账户 4 个 toggle(ADR-0026 §9)

v4 之前只说 1 个 toggle(includeInNetWorth)。实际是 4 个:includeInNetWorth(开) / isPinned(特别关注,置顶) / isDefaultIncomeAccount(默认收账) / isDefaultExpenseAccount(默认支出)。

「退款」按钮 = 撤销最近一笔交易(ADR-0026 §9)

咔皮图 10 显示「删除/退款」2 按钮。「退款」不是「撤销账户所有交易」,而是「撤销最近一笔关于该账户的交易」。需 transactionDao 新增 reverseLastTransaction(accountId) 方法。

资产 vs 负债 分类(ADR-0026 §6)

v4 之前净资产只在 §22「净资产仪表盘」设计。但资产/负债分类要在 S02 就实现,否则后续没法算。净资产 = 资产 - 负债,显式拆分显示。

添加账户弹层 v4 漏掉的设计(ADR-0026 §10)

  • v4 之前说「6 种账户类型」→ 实际4 大类 × 17+ 子类(列表见上)
  • 每种都标「(资产)/(负债)」明确
  • 每个子类有真实品牌 emoji + 名称
  • 各大类有「自定义」选项

账户详情 / 余额变动明细(ADR-0026 §8)

2 tab:「记账 / 转账」分类,每月统计(流出/流入),每笔带分类 + 备注 + 金额 + 所属账户。v4 之前只说「账户详情」,没提 2 tab 分类。

收入分类 8 子类(ADR-0026 §10 补充)

v4 之前没列收入分类。实际是 8 子类:职业收入 / 经营收入 / 保险理财 / 资金往来 / 二手买卖 / 好运收入 / 生活费 / 其他收入。

class AuditService {
  AuditResult audit(Transaction tx) {
    final results = <AuditResult>[];

    // 规则 1: 金额异常
    if (await _checkAmountAnomaly(tx)) {
      results.add(AuditResult.red('单笔超过日均 5 倍'));
    }

    // 规则 2: 时间异常
    if (await _checkTimeAnomaly(tx)) {
      results.add(AuditResult.yellow('深夜餐饮'));
    }

    // 规则 3: 预算异常
    if (await _checkBudgetAnomaly(tx)) {
      results.add(AuditResult.orange('预算即将超支'));
    }

    // 规则 4: 重复
    if (await _checkDuplicate(tx)) {
      results.add(AuditResult.confirm('检测到可能的重复'));
    }

    // 规则 5: 限额
    if (await _checkCreditLimit(tx)) {
      results.add(AuditResult.red('信用卡额度告急'));
    }

    return AuditResult.combine(results);
  }
}

  const Success(this.data);

  final Failure failure;
  const Failure(this.failure);
}

Future<Result<Transaction>> add(Transaction tx) async {
  try {
    final id = await _dao.insert(tx);
    return Success(tx.copyWith(id: id));
  } on DatabaseException catch (e) {
    return Failure(DatabaseFailure(e.message));
  }
}

# .github/workflows/ios-build.yml
name: iOS Build

on:
  push:

jobs:

    runs-on: macos-latest
    timeout-minutes: 30

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Flutter
        uses: subosito/flutter-action@v2
        with:
          flutter-version: '3.24.5'
          channel: stable

      - name: Cache pub
        uses: actions/cache@v4
        with:
          path: |
            ~/.pub-cache
            build/
            .dart_tool/
          key: ${{ runner.os }}-pub-${{ hashFiles('**/pubspec.lock') }}

      - name: Get dependencies
        run: flutter pub get

      - name: Run code generation
        run: dart run build_runner build --delete-conflicting-outputs

      - name: Analyze
        run: flutter analyze

      - name: Run tests
        run: flutter test --coverage

      - name: Setup Xcode
        uses: maxim-lobanov/setup-xcode@v1
        with:
          xcode-version: latest-stable

      - name: Install Apple Certificate
        env:
          BUILD_CERTIFICATE_BASE64: ${{ secrets.BUILD_CERTIFICATE_BASE64 }}
          P12_PASSWORD: ${{ secrets.P12_PASSWORD }}
          KEYCHAIN_PASSWORD: ${{ secrets.KEYCHAIN_PASSWORD }}
        run: |
          # 创建临时 keychain
          KEYCHAIN_PATH=$RUNNER_TEMP/build.keychain
          security create-keychain -p "$KEYCHAIN_PASSWORD" $KEYCHAIN_PATH
          security set-keychain-settings -lut 21600 $KEYCHAIN_PATH
          security unlock-keychain -p "$KEYCHAIN_PASSWORD" $KEYCHAIN_PATH
          # 导入证书
          echo "$BUILD_CERTIFICATE_BASE64" | base64 --decode > cert.p12
          security import cert.p12 -k $KEYCHAIN_PATH -P "$P12_PASSWORD" -T /usr/bin/codesign
          # 设置搜索路径
          security list-keychain -d user -s $KEYCHAIN_PATH
          security set-key-partition-list -S apple-tool:,apple: -k $KEYCHAIN_PASSWORD $KEYCHAIN_PATH

      - name: Install Provisioning Profile
        env:
          PROVISIONING_PROFILE_BASE64: ${{ secrets.PROVISIONING_PROFILE_BASE64 }}
        run: |
          echo "$PROVISIONING_PROFILE_BASE64" | base64 --decode > profile.mobileprovision
          mkdir -p ~/Library/MobileDevice/Provisioning\ Profiles
          cp profile.mobileprovision ~/Library/MobileDevice/Provisioning\ Profiles/

      - name: Build iOS (Release)
        run: |
          flutter build ipa --release \
            --export-options-plist=ios/ExportOptions.plist

      - name: Upload .ipa artifact
        uses: actions/upload-artifact@v4
        with:
          name: app-release-ipa
          path: build/ios/ipa/*.ipa
          if-no-files-found: error

      - name: Upload .app artifact
        uses: actions/upload-artifact@v4
        with:
          name: app-release-app
          path: build/ios/iphoneos/Runner.app
          if-no-files-found: warn

      - name: Cleanup keychain
        if: always()
        run: |
          KEYCHAIN_PATH=$RUNNER_TEMP/build.keychain

class PerfMonitor {
  static final _logs = <PerfLog>[];

  static Future<T> measure<T>(
    String name,
    Future<T> Function() action, {
    Map<String, dynamic>? metadata,
  }) async {
    final stopwatch = Stopwatch()..start();
    try {
      return await action();
    } finally {
      stopwatch.stop();
      if (stopwatch.elapsedMilliseconds > 100) {
        _logs.add(PerfLog(
          name: name,
          durationMs: stopwatch.elapsedMilliseconds,
          metadata: metadata,
          at: DateTime.now(),
        ));
      }
    }
  }
}

  () => repo.getAllInMonth(...),

52.3 优先级决策原则

  1. 使用频率：每天用 > 每周用 > 偶尔用
  2. 痛点强度：解决 vs 改善 vs 锦上添花
  3. 技术债务：先还债再加新
  4. 成本：低 ROI 不做
  5. 个人喜好：能让我自己更愿意用

52.4 技术债务管理

  • 数据库查询未加索引
  • Repository 内逻辑过多（应拆 Service）
  • Widget 内嵌业务逻辑（应拆 Notifier）
  • 硬编码字符串 / 颜色
